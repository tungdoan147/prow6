Docker

1.  Khái niệm Docker
- Là một công cụ giúp cho việc tạo và triển khai container để phát triển . Các container là môi trường, ở đó lập trình viên đưa thành phần cần thiết để ứng dụng chạy được
- Docker trông khá giống máy ảo tuy nhiên nó có những điểm khác như: thay vì tạo ra toàn bộ hệ thống ( dù ảo hóa ), Docker cho phép sử dụng nhân của hệ điều hành đang chạy Docker để chạy ứng dụng bằng cách bổ sung thêm các thành phần còn thiếu cung cấp bởi container. Cách này sẽ làm tăng hiệu suất và giảm kích thước ứng dụng

2. Các lệnh

`docker --version` - kiểm tra phiên bản docker đang sử dụng 
`docker info` - ktra thông tin docker

- image là một gói phần mềm trong đó chứa những thứ cần như thư viện, các file cấu hình, biến môi trường để chạy ứng dụng nào đó ( như usb chưas hệ điều hành win )
- Khi một phiên bản image chạy, phiên bản đó gọi là container => muốn có container phải có image 

`docker images -a` - kiểm tra images đang có

- Repository là tên images trên kho chứa. TAG là phiên bản image . IMAGE ID chuỗi định danh duy nhất

`docker pull nameimage:tag` - tải images 

`docker ps ` - kiểm tra các container nào đang chạy
`docker start containerid` bật container
`docker run [OPTIONS] IMAGE [COMMAND] [ARG]` tạo và chạy 1 docker. OPTIONS-các thiết lập khi tạo container IMAGES tên images [COMMAND] [ARG] lệnh tham số container chạy
VD `docker run -it ubuntu` (it) để khi chạy có terminate làm việc ngay với ubuntu

`docker run -it --rm ubuntu` tạo , chạy container, container tự xóa khi kết thúc

`docker container attach containerid ` vào container đang chạy

`docker container start -i containerid ` chạy 1 container đăng dừng

`docker container ra containerid ` xóa container



- Mount đường dẫn file docker container

`docker run -itd --name name --mount type=bind,src=/OPT/test,dst=/opt/ ubuntu`

( Chú ý : ubuntu là tên images name là tên container tạo . Đường dẫn phải tồn tại giống destination )
